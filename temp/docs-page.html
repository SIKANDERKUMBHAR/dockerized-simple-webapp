InfraFlowOps Overview

InfraFlowOps automates the provisioning of AWS EC2 instances, SSL configuration for Nginx, and deployment of Dockerized applications using CI/CD pipelines. It's designed to simplify the management of DevOps workflows in cloud infrastructure.

Features
  • Automates EC2 instance provisioning using AWS Boto3.
  • Configures SSL certificates for Nginx with ssl_conf.py.
  • Builds, tags, and pushes Docker images to Docker Hub or AWS ECR.
  • Deploys Dockerized applications on EC2 instances.
  • Enables CI/CD operations with cicd.py for seamless updates to deployed applications.
File Descriptions
1. autoinfra.py

This script automates infrastructure provisioning and application deployment. It provisions EC2 instances, sets up Nginx and Docker, builds Docker images, pushes them to the specified registry, and deploys the Dockerized application on EC2 instances.

  • plan_resources() - Simulates resource creation without execution.
  • create_ec2_instance() - Creates an EC2 instance and returns the public IP.
  • docker_login() - Authenticates with Docker Hub or AWS ECR.
  • build_and_tag_docker_image() - Builds and tags the Docker image.
  • pull_and_run_docker_image() - Pulls and runs the Docker container on the EC2 instance.
  • terminate_ec2_instance() - Terminates the EC2 instance.
2. ssl_conf.py

This script automates the SSL configuration for Nginx. It transfers SSL certificate and private key files to the EC2 instance and updates the Nginx configuration to enable HTTPS, while redirecting HTTP traffic to HTTPS.

  • get_public_ip() - Reads the EC2 public IP from a file.
  • transfer_ssl_files() - Transfers SSL files to the EC2 instance and sets proper permissions.
  • update_nginx_config() - Configures Nginx for HTTPS with the SSL certificate and private key.
3. cicd.py

This script handles the CI/CD operations, automating Docker login, build, and push operations. It updates Docker containers on EC2 instances by removing outdated containers and images.

  • docker_login() - Logs in to the container registry.
  • build_and_tag_docker_image() - Builds and tags Docker images for deployment.
  • push_docker_image() - Pushes Docker images to the registry.
  • update_docker_container() - Updates the Docker container on the EC2 instance.

Setup

Requirements
  • Python 3.x
  • AWS CLI configured with proper credentials
  • Docker installed locally
  • Required Python packages: pip install boto3 paramiko python-dotenv
Environment Variables

		SUBDOMAIN=your-subdomain.example.com
		DOCKERHUB_USERNAME=your-dockerhub-username
		DOCKERHUB_PASSWORD=your-dockerhub-password
		KEY_FILE_PATH=/path/to/your/aws-private-key.pem
		LOCAL_SSL_CERT=/path/to/ssl-certificate.crt
		LOCAL_SSL_KEY=/path/to/ssl-private-key.key
		REGISTRY_TYPE=dockerhub  # or 'ecr' for AWS Elastic Container Registry
		CODE_DIR=/path/to/your/code/directory

Project Workflow

The typical project workflow is as follows:

  • Provision EC2 using autoinfra.py, which sets up Nginx and Docker on the EC2 instance.
  • Deploy SSL using ssl_conf.py to transfer SSL certificates and configure Nginx for HTTPS.
  • Build and push Docker images to the registry, and deploy them on the EC2 instance using autoinfra.py.
  • Handle future updates with the cicd.py script, ensuring smooth CI/CD delivery of applications.

Logs and Troubleshooting

Logs are generated by each script for debugging:

  • Nginx Logs: /var/log/nginx/
  • Docker Logs: docker logs
  • AWS Security Groups: Ensure ports 80 and 443 are open.
  • SSL Configuration: Verify certificates are valid using OpenSSL: openssl x509 -in .crt -text -noout

FAQs

Below are some frequently asked questions about InfraFlowOps, a Python-based automation tool designed to streamline DevOps workflows.

General Questions

What is InfraFlowOps?

InfraFlowOps is a Python-based automation tool that provisions AWS EC2 instances, configures SSL certificates for Nginx, and deploys Dockerized static websites using CI/CD pipelines. It simplifies infrastructure management and deployment processes for DevOps workflows.

Who should use InfraFlowOps?

InfraFlowOps is ideal for developers and DevOps engineers looking to automate AWS infrastructure setup, manage SSL configurations, and streamline Docker-based application deployments with CI/CD integration.

Technical Details

What AWS services does InfraFlowOps support?

InfraFlowOps supports provisioning AWS EC2 instances, managing Nginx SSL configurations, and deploying Dockerized static websites through AWS resources.

Can InfraFlowOps integrate with existing CI/CD pipelines?

Yes, InfraFlowOps is designed to work seamlessly with CI/CD pipelines to automate the deployment of static websites in Docker containers.

Is it possible to customize SSL certificate configurations?

Absolutely. InfraFlowOps allows you to configure and manage custom SSL certificates for your Nginx server.

Troubleshooting

What should I do if an EC2 instance fails to provision?

Check your AWS credentials, permissions, and quota limits. Ensure the InfraFlowOps configuration file has the correct details for the desired region and instance type.

How can I debug deployment issues with Dockerized websites?

Verify the Docker image and container logs for errors. Ensure that the Dockerfile is correctly configured and that all required environment variables are set.

What are common SSL configuration errors?

Ensure that the SSL certificate files are correctly specified in the Nginx configuration. Check for proper file permissions and validate the SSL certificate and private key.
